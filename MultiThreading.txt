CPU: it is a processing unit 
It has cores inside it, previously there is only 1 code inside cpu but now a days it has multiple cores 
like a quad core cpu has 4 cores, for an instance once core could hand your web browser another music player and so on.

Program: set of instruction written in a particular programming language 
eg: ms word

Process:  it is an instance of program, when we run a program os start the process for that program 

Thread: It is the smallest unit of execution with in the process. A process can have multiple threads, which share the same resources but can run independently 
eg: chrome using multiple threads for different tabs 

Multitasking:
allows os to run multiple processes simultaneously. on single core this is done through time sharing, rapidly switching between tasks. on multi core true parallel execution occurs with tasks distributed across cores.
managing resources between the completely sperate programs .

Multithreading :
it enhances the efficiency of Multitasking by breaking down individual tasks into smaller subtasks or threads.  These threads process simultaneously making use of the cpu capabilities.
managing resources within a single process

eg: web browser using different threads for downloading playing video etc.


In single core system both threads and processes are managed by os schedular through time slicing and context switching to create a illusion of simultaneous execution.

In multi core systems:
both threads  and processes can run in true parallel on different cores, with the os scheduler distributing tasks across the cores to optimize performance 

Time Slicing-> divides the cpu time into small intervals called time slices  or quanta.
os scheduler allocate the time slice to diff process or threads 

Context Switching -> saving the state of currently running process and loading the state of next process or thread.
when process's Time slice expires os scheduler performs the context switching to move the cpu focus to another process or thread by pausing the current one

Java Deadlock:
deadlock is a situation in where threads got stuck waiting for each other 
Conditions for deadlock:
1)Mutual Exclusion: only one thread can access the thread at a time 
2)Hold and wait
3)No preemption
4)circular wait: a set of thread is waiting for each other in circular chain 

ex: let say there are 2 threads both wants to write one holding (locking) pen and waiting for paper while other holding(locking) paper waiting for pen.

Fail Fast and Fail Safe:
Fail Fast:
Iterates over the original collection.
If the collection is modified during iteration (except using Iterator.remove()), it throws ConcurrentModificationException.
Faster because it doesn't create a copy.
Examples: ArrayList, HashMap, HashSet.

Fail Safe:
Iterates over a copy (snapshot) of the collection.
Modifying the original collection during iteration does not throw an exception.
Slightly slower and uses more memory because it creates a copy.
Examples: CopyOnWriteArrayList, CopyOnWriteArraySet, ConcurrentHashMap (weakly consistent iterator).
