Internal Working of HashMap:
Basic components of hashmap 
1)Key-> identifier used to retrieve the value
2)value-> The data associated with key 
3)Bucket-> A place where key value pairs are stored. Think of bucket as cells in a list or array 
4)Hash Function -> converts a key to index (bucket location or storage)
	Hash function is an algorithm that takes an input or key and returns a fixed sized string of bytes, typically a numerical value.
	The output is known as hash code, hash value or simply hash. The primary purpose of a hash function is to map data of arbitrary size to data of fixed size.
Characteristics of hash function:
Deterministic: same input will always produce same output 
Fixed output size: regardless of output size code has consistent size (eg: 32but, 64 bit)
Efficient Computation: 

How data stored in hashMap:
1) Hashing the Key-> key passed through hash function to generate unique hashcode (Pans integer number). this hash code helps determine where the key value pair will be stored in the array (bucket array).

2)Calculating the index: hash code then used to calculate an index int the array (bucket location) using :
int index= hashcode % arraySize;
this index decides which bucket will store the key value pair 

By default the size of array is 16 
3) Storing in the bucket:
key-value pair stored in the bucket at the calculated index.
Each bucket can hold multiple key-value pairs 
this is called collision handling mechanism****
Handling Collision:
Since different keys can generate same index(called collision) hashmap uses technique to handle this situation. 

[ on every index of bucket array linked list is stored but in java 8 we use balanced binary search tree or RB tree( as linked list uses linear search to search the element and it takes a lot of time), in case of collision it has pointer to next node or if multiple  kv pairs map to the same bucket
basically what happen the linkedlist get converted to red black tree after certain threashold (default 8) thereashhold is called as Treefy Threashhold
time complexity in case if RB tree -> O(logn )

when they retireieved hashmpa traverse the linked list of bst untill it founds the match 

How data is retrieved from hash map :
1)hashing the key-> similar to insertion 
2) finding the index-> hash code is used to find the index of the bucket where the key value pair is stored 
3)Searching the bucket -> once the correct bucket is found it checks for the key in that bucket. if it finds the key it return the associated value .


HashMap Resizing(Rehashing):
HashMap has an internal array size, which is 16 by deafult when the numeber of elements (key-value) pair grows to certain load factor(defualt 0.75) hashmap automatically resizes the array to hold more data. This is called Rehashing.

The default array size is 16 so when more than 12 elements (16*0.75) are inserted, the hashmap will resize.

In hashmap constructor as well we can give initial capacity as and load factor  <IC, LF>
All entries are rehashed (i.e their positions are recalculated) and placed into the new array 

Time Complecity: O(1) without coliision, with collsion O(n) with balanced BST it is O(logn).
in case of constiansKey: TC same 
in case of caontinsValue: TC O(n)  for both (WC and BC)
size -> O(1) for all

in case of collitiosn equals methods is used ti check the index match
